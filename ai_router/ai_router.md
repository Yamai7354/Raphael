# AI Router - Autonomous Multi-Agent Orchestration Kernel

The **AI Router** is a sophisticated system designed to orchestrate AI tasks across heterogeneous compute clusters. It has evolved from a simple request router into a self-optimizing, autonomous kernel capable of managing complex multi-agent workflows.

## 🚀 Key Features

*   **Smart Routing**:  
    *   **Role-Based**: Directs tasks to `coder`, `chat`, or `summarizer` specialists.
    *   **Capability-Aware**: Matches model size (e.g., 70B vs 7B) to node VRAM capabilities.
    *   **Federated**: Routes tasks across multiple clusters or cloud providers.

*   **Advanced Execution**:  
    *   **Task Graphs**: Supports complex DAGs with dependencies.
    *   **Async Scheduler**: Non-blocking execution for high throughput.
    *   **Predictive Planning**: Prefetches resources based on predicted future tasks.

*   **full Autonomy**:  
    *   **Self-Healing**: Circuit breakers and automatic failover.
    *   **Optimization**: Continuously learns from execution metrics to improve routing.
    *   **Meta-Planning**: Autonomously generates maintenance and optimization tasks.

## 📦 Installation & Setup

### Prerequisites
*   Python 3.9+
*   `pip`
*   Define your cluster capabilities in `config.json` (optional, defaults provided).

### Quick Start

1.  **Clone the repository**:
    ```bash
    git clone <repo-url>
    cd ai-router
    ```

2.  **Install dependencies**:
    ```bash
    python3 -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt
    ```

3.  **Run the Server**:
    ```bash
    python main.py
    ```
    Server will start at `http://localhost:9000`.

## 🛠️ Usage

### 1. Register a Node (Worker)
Workers register themselves with capabilities:
```bash
curl -X POST http://localhost:9000/register \
  -H "Content-Type: application/json" \
  -d '{
    "node_id": "worker-1",
    "capabilities": {
      "vram": 24576,
      "supported_models": ["llama-3-8b", "mistral-7b"]
    }
  }'
```

### 2. Submit a Task
Submit a task to the router. The system will auto-select the best node.
```bash
curl -X POST http://localhost:9000/route \
  -H "Content-Type: application/json" \
  -d '{
    "task_id": "task-1",
    "role": "coder",
    "priority": "high",
    "context_size": 4096,
    "prompt": "Write a Python script to..."
  }'
```

### 3. Create a Multi-Agent Workflow
Define a DAG of steps. The Supervisor will manage execution.
```bash
curl -X POST http://localhost:9000/admin/workflows/create \
  -H "Content-Type: application/json" \
  -d '{
    "workflow_id": "wf-1",
    "steps": [
      {"id": "s1", "role": "chat", "description": "Discuss requirements"},
      {"id": "s2", "role": "coder", "description": "Implement code", "dependencies": ["s1"]}
    ]
  }'
```
